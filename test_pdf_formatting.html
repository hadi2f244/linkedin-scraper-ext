üîç Root Cause Analysis

Problem 1: The PDF had an unwanted <h1>Cover Letter</h1> heading

Solution: Removed the h1 tag from the HTML template


Problem 2: Cover letter text was stored as plain text (textContent) but PDF generation was using innerHTML

Why it failed: Plain text newlines (\n) are ignored by HTML rendering

Solution: Convert plain text to HTML by:

  • Splitting on double newlines (\n\n) to identify paragraphs
  • Wrapping each paragraph in <p> tags
  • Converting single newlines to <br> tags within paragraphs

üíª Code Changes

After (Lines 80-139):
